# Product

## Register

product

## Users

A3 Study Agent serves university learners, soon-to-be university students, and demo reviewers who need to see a trustworthy learning assistant in action. Learners use it while clarifying course concepts, generating study resources, building learning plans, or asking for academic support. Reviewers use the interface to understand both the user-facing answer and the agent execution path behind it.

## Product Purpose

The product is a university learning resource generation workspace. It combines chat, local course-material retrieval, web evidence, resource artifacts, and execution observability so learners can turn fuzzy academic needs into concrete explanations, mindmaps, exercises, review documents, and study plans.

Success means the learner can quickly understand what the assistant can do, ask a course or planning question, follow the generation progress without confusion, recover from errors, and reuse prior conversations. The interface should make complex agent behavior feel legible rather than mysterious.

## Brand Personality

Calm, capable, and academically grounded. The product should feel like a focused study desk with a transparent AI control panel attached: warm enough for learners under pressure, precise enough for technical demos, and restrained enough for repeated use.

## Anti-references

Avoid generic AI chatbot spectacle, purple-blue gradient SaaS styling, decorative glassmorphism, empty marketing-card layouts, unreadable tiny status text, and playful visuals that make the system feel less reliable. Avoid UI copy that sounds like a landing page instead of a working learning tool.

## Design Principles

1. Make state visible. Every running, waiting, completed, controlled-stop, and error state must be understandable at a glance.
2. Keep the workbench calm. Use restrained color, consistent surfaces, and predictable controls so the learner can focus on the task.
3. Explain enough, not everything. The UI should translate agent steps into human language without overwhelming the chat.
4. Preserve learning context. Conversation history, resource artifacts, and node trails should feel connected, not like separate panels.
5. Respect the legacy module boundary. The volunteer page may share the visual system, but its route, storage, and business behavior remain independent.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Prefer clear Chinese labels, stable focus states, keyboard-reachable controls, non-color-only status indicators, and reduced-motion-safe transitions. Text must not overflow compact panels, graph nodes, buttons, or chat cards.
